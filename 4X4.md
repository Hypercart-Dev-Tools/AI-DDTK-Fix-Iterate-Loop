# About the Current Project
AI-DDTK (AI Driven Development ToolKit) is a central, install-once toolkit used across WordPress projects to standardize AI agent guidance, security practices, and developer workflows (e.g., WPCC scanning, runtime perf profiling, AJAX endpoint testing).

This week’s focus is improving resilient AI-agent execution with an optional tmux proxy layer while keeping install/status output and docs aligned.

## 4. Recent Lessons Learned
**Capture insights to improve processes and avoid repeating mistakes.**

1. AI agents need a terminal recovery path: persistent sessions matter when IDE terminals become non-responsive.
2. Explicit binary paths are safer than shell-init assumptions when commands run under different shells.
3. Documentation should mirror operational reality: install/status output, README guidance, and agent instructions must stay in sync.
4. Keep dashboard history lightweight when the changelog already records completed work in detail.
