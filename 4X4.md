## 4. Recent Lessons Learned
**Capture insights to improve processes and avoid repeating mistakes.**

1. Never use `eval` with user-controlled strings in shell scripts — bash arrays with `"${arr[@]}"` invocation are safer and just as flexible.
2. `.mjs` files run as ESM — `require()` is CJS-only. Use `.js` extension for scripts that need `require()`.
3. README claims should match implementation reality — review auth paths, module resolution (including global npm / `NODE_PATH` recovery), encoding, and PHP version requirements before documenting a feature as ready.
4. Auth verification should check multiple signals (cookies, redirect behavior, error pages) not just "did we avoid the login page".
