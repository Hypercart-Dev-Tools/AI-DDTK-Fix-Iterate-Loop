# Changelog\n\nAll notable changes to the AI-DDTK VS Code Extension will be documented in this file.\n\nThe format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),\nand this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).\n\n## [0.1.0] - 2026-03-26\n\n### Added\n\n- Initial experimental release of AI-DDTK VS Code Extension\n- **Status Bar Integration** — Real-time AI-DDTK readiness indicator with contextual icons\n- **Project Wiring** — One-click MCP configuration setup via `wire-project` script\n- **Preflight Checks** — Verify AI-DDTK installation and dependencies\n- **WPCC Scanning** — Run WordPress security/performance scans from the editor\n  - Scan current file\n  - Scan entire project\n  - Scan custom path\n- **Documentation Access** — Quick links to AI-DDTK docs and GitHub repository\n- **Auto-Detection** — Automatically detects WordPress projects and offers setup\n- **Configuration** — Customizable settings for paths, auto-wire, status bar, and MCP server\n- **Command Palette Integration** — All features accessible via VS Code command palette\n- **TypeScript Support** — Full TypeScript implementation with strict mode\n- **ESLint Configuration** — Code quality checks and linting\n\n### Features\n\n#### Commands\n\n- `ai-ddtk.wireProject` — Set up MCP configuration for current workspace\n- `ai-ddtk.runPreflight` — Verify AI-DDTK installation and dependencies\n- `ai-ddtk.runWpcc` — Run WordPress code security/performance scan\n- `ai-ddtk.openDocs` — Access AI-DDTK docs and GitHub repo\n- `ai-ddtk.checkStatus` — Display AI-DDTK installation status\n\n#### Status Bar States\n\n- 🔴 **Error** — AI-DDTK not installed\n- 🟡 **Setup** — WordPress project detected, not yet wired\n- 🟢 **Ready** — MCP configured and ready to use\n- 🔵 **Info** — AI-DDTK installed, non-WordPress workspace\n\n#### Configuration Options\n\n- `ai-ddtk.aiDdtkPath` — Path to AI-DDTK installation (default: `~/bin/ai-ddtk`)\n- `ai-ddtk.autoWireOnOpen` — Auto-wire WordPress projects on open (default: `false`)\n- `ai-ddtk.showStatusBar` — Show status bar indicator (default: `true`)\n- `ai-ddtk.enableMcpServer` — Enable MCP server integration (default: `true`)\n\n### Technical Details\n\n- **Activation Events** — `onStartupFinished`, WordPress project detection, command triggers\n- **Build System** — esbuild for fast bundling and minification\n- **Target** — VS Code 1.85.0+, Node.js 18+\n- **Language** — TypeScript with strict mode\n\n### Known Limitations\n\n- Extension is experimental and subject to change\n- Requires AI-DDTK to be installed at `~/bin/ai-ddtk`\n- MCP server integration requires `wire-project` script from experimental folder\n- Status detection relies on file system checks (wp-config.php, wp-content/, etc.)\n\n### Future Enhancements\n\n- [ ] Marketplace publication\n- [ ] Settings UI for easier configuration\n- [ ] Integration with VS Code's built-in terminal\n- [ ] Real-time WPCC scan results in editor\n- [ ] Query Monitor profiling integration\n- [ ] Playwright auth integration\n- [ ] Project templates and scaffolding\n- [ ] Multi-workspace support\n- [ ] Custom keybindings\n\n---\n\n## Development\n\nSee [README.md](./README.md) for development setup and contribution guidelines.\n"
